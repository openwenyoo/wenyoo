# 编辑器工作流

## 构建故事图谱

图谱画布适合做可视化创作，但最终显示出来的连线依赖故事中的 `connections`。

推荐循环：

1. 新建或加载故事。
2. 添加或修改节点。
3. 编辑动作、触发器、物品和角色。
4. 保存。
5. 需要时重新编译连接图谱。
6. 在游戏界面里试玩。

## 处理节点

节点用于表示地点、场景或状态切换。

建议重点检查这些字段：

- `name`
- `definition`
- `explicit_state`
- `implicit_state`
- `objects`
- `actions`
- `triggers`

编辑器提供可视化界面，但保存后的数据仍然会映射到规范 YAML 字段。

## 编辑动作、物品和触发器

编辑器为嵌套内容提供了二级编辑流程：

- 动作
- 物品
- 触发器

这样可以在不让主图过于拥挤的前提下编辑结构化数据。

编写新内容时，优先使用规范字段：

- 动作文案用 `text`，不要继续沿用旧的 `name`
- 效果使用 `type`
- 导航和变量路径在 schema 要求的位置使用 `target`

## 角色、参数、Lore 与全局物品

通过浮动工具面板可以打开更高层级的故事数据：

- 角色
- 参数和变量
- lore
- 全局物品

当单纯看节点图不足以理解世界状态时，这些面板很有帮助。

## 编译连接图谱

连接图状态控件会提示当前图谱是：

- current
- stale
- missing

以下情况建议重新编译：

- 修改了导航
- 修改了物品关系
- 修改了角色位置
- 图谱显示和故事逻辑不一致

编辑器会调用与 `tools/compile_connections.py` 相同的编译流程。

## 保存策略

编辑器很适合快速迭代，但要理解保存行为：

- 保存会写入一个扁平化后的故事表示
- 原本依赖多文件 `includes` 的故事，经过编辑器往返后需要额外确认
- 如果你的工作流依赖保持文件拆分，请在保存后认真检查输出，再提交

## 什么时候切回 YAML

以下情况建议直接编辑 YAML：

- 故事高度模块化
- 需要批量修改
- 需要更精确地控制 schema
- 需要查看生成的 `connections` 或高级 forms/functions

编辑器和 YAML 不是互斥关系，而是互补关系。
