# 编辑器总览

可视化编辑器默认地址是 `http://localhost:8000/editor`。

## 适合谁使用

以下场景适合使用编辑器：

- 想通过可视化节点图创作
- 想在浏览器里加载和查看故事
- 需要内置的保存到服务器能力
- 想使用版本历史
- 想使用 AI 辅助编辑和导入流程

以下场景更适合直接写 YAML：

- 需要精确控制多文件结构
- 想在代码编辑器里做批量重构
- 需要更清晰的代码审查 diff
- 想直接对照权威格式文档工作

## 编辑器擅长什么

- 从服务端加载故事
- 可视化节点和已编译的连接关系
- 编辑节点、动作、触发器、物品、角色、参数、lore 和全局物品
- 创建新故事
- 把故事保存回仓库
- 在 `saves/story_versions/` 下保留备份版本
- 在 UI 中触发连接图谱编译

## 关键行为

- 编辑器通过后端加载故事，并会先合并 `includes`
- 保存时会把当前合并后的故事写回入口故事路径
- 图谱显示依赖故事里的 `connections`，所以当结构变化后图没有更新时，需要重新编译连接图谱

## 当前边界

- 它不是完整的内嵌游戏试玩器
- 浏览器内导出 YAML 不是主要保存路径，正常保存依赖服务端 API
- 旧文档里提到的单一 `editor-guide.md` 已被这一组编辑器文档替代
