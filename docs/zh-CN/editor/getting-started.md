# 编辑器快速上手

## 打开编辑器

1. 用 `python -m src.main` 启动服务。
2. 打开 `http://localhost:8000/editor`。
3. 等待加载面板出现。

## 首屏说明

加载面板是编辑器的正常入口。在这里你可以：

- 创建新故事
- 加载已有的服务端故事
- 打开导入向导

## 主要区域

- 菜单栏：文件操作、视图模式、撤销/重做、语言、连接图状态
- 图谱画布：节点布局和连接关系
- 右侧编辑区：节点详情，以及动作、触发器、物品等二级编辑器
- 浮动工具面板：角色、参数、lore 和全局物品

## 新建故事

1. 选择新建流程。
2. 输入故事标题。
3. 编辑器会创建一个起始节点。
4. 保存后，故事才会真正写入 `stories/`。

在保存之前，新故事只存在于编辑器当前状态中。

## 加载已有故事

- 可以从加载面板或菜单栏中的故事列表打开故事。
- 编辑器会向后端请求故事数据。
- 如果故事使用了 `includes`，后端会先合并后再返回给编辑器。

## 保存故事

- 保存通过 `/api/story/{story_id}` 把当前故事写入服务端。
- 后端会在 `saves/story_versions/` 下保存版本备份。
- 保存后编辑器会重新加载故事，并清除未保存状态。

## 导入说明

导入流程更适合做辅助转换和大纲生成，而不是简单的 YAML 文件双向同步。

请把导入结果当作起点，在保存前认真检查生成出的图谱内容。

## 第一次安全使用流程

1. 新建或加载一个故事。
2. 只修改少量节点。
3. 保存一次。
4. 如果连接图状态显示缺失或过期，重新编译一次连接图谱。
5. 在另一个标签页打开玩家界面进行试玩。
